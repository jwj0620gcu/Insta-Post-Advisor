**5명의 AI 전문가** | **874건 실제 데이터 학습** | **3라운드 토론 기반 진단**

## 왜 InstaRx인가

| | 기존 툴 | InstaRx |
|---|---|---|
| **점수 근거** | 주관적 경험 / 단일 모델 채점 | 874건 실제 게시물 회귀 분석 → 8개 카테고리 차별화 가중치 |
| **진단 방식** | 단일 LLM 1회 호출 | 5 에이전트 병렬 진단 → 교차 반박 토론 → 심판 종합 |
| **제안 품질** | "캡션 매력도를 높이세요" | "캡션 「XX」→ 「5분 완성! 엄마 20년 비법 레시피」→ 숫자+감정+궁금증" |
| **댓글 예측** | 없음 | AI가 실제 인스타 스타일로 댓글 시뮬레이션 |
| **최적화 루프** | 제안만 제공 | 고득점 개선안 3개 자동 생성 + 즉시 재채점 |
| **데이터 기반** | 없음 | Spearman 상관 · 선형 회귀 · K-Means 클러스터링 · LLM 심층 분석 |

## 주요 기능

- **멀티모달 입력**: 이미지·릴스·스크린샷 드래그앤드롭 / Ctrl+V 붙여넣기, AI가 제목·본문·카테고리 자동 인식
- **실시간 진단 애니메이션**: 11단계 타임라인 + 토론 실황 말풍선 + 에이전트 상태 추적
- **5차원 레이더 채점**: 콘텐츠 품질 · 비주얼 · 성장 전략 · 인게이지먼트 잠재력 · 종합 점수
- **AI 시뮬레이션 댓글**: 실제 인스타 스타일(논쟁·의문·대댓글 포함), 좋아요 수 예측
- **반복 최적화 엔진**: 고득점 개선안 3개 자동 생성, 재채점 후 최고점 방안 추천
- **베이스라인 비교**: 동일 카테고리 수천 건 게시물과 비교(캡션 길이 / 해시태그 수 / 바이럴률)
- **진단 히스토리**: 로컬 IndexedDB 저장, 개인정보 안전

## 기술 아키텍처

### 3대 자체 학습 모델

| 모델 | 학습 데이터 | 역할 |
|---|---|---|
| **Model A — 정량 예측 엔진** | 874건 실제 게시물 · 회귀 분석 | 8개 카테고리 차별화 가중치 · 5차원 즉시 채점 · LLM 없이 50ms 이내 |
| **Baseline Knowledge Graph** | 874건 게시물 + 2,465건 댓글 · K-Means 클러스터링 | 카테고리별 바이럴 기준선 · 인게이지먼트 중앙값 · 해시태그 분포 · 게시 시간대 히트맵 |
| **Comment Persona Engine** | 2,465건 실제 댓글 · LLM 분류 | 6종 유저 페르소나 · 감정 분포 · 좋아요 예측 |

### 4단계 진단 파이프라인

```
Stage 1                     Stage 2                 Stage 3                     Stage 4
데이터 기반 기준선 학습  →   Model A 즉시 채점   →   멀티 에이전트 심층 토론   →   AI 최적화 루프

874건 + 2,465 댓글          5차원 즉시 채점          4 에이전트 병렬 진단          개선안 3개 자동 생성
Spearman / 회귀 / 클러스터   50ms · LLM 없음          교차 반박 · 근거 보강         즉시 재채점
8개 카테고리 가중치          카테고리별 기준선          심판 에이전트 최종 종합         최고점 방안 추천
```

### 멀티 에이전트 토론 구조

```
Round 1: 병렬 진단                    Round 2: 교차 토론                  Round 3: 종합 판정

[콘텐츠 분석가] ─┐                    콘텐츠 ←→ 비주얼                    ┌─→ 최종 점수
[비주얼 진단가] ─┤→ 독립 진단 + 채점  비주얼  ←→ 성장      반박/보충      ├─→ 캡션·해시태그 최적화
[성장 전략가]  ─┤                    성장   ←→ 사용자                    ├─→ 비주얼 개선 방향
[사용자 시뮬]  ─┘                    사용자 ←→ 콘텐츠                    └─→ 시뮬레이션 댓글
```

## 빠른 시작

```bash
# 클론
git clone https://github.com/jwj0620gcu/Insta-Post-Advisor.git
cd Insta-Post-Advisor

# 환경 변수 설정
cp .env.example backend/.env
# backend/.env 에서 API 키 입력

# 의존성 설치 + DB 초기화 + 서버 실행
make install && make data && ./start.sh
```

브라우저에서 `http://localhost:5173` 접속

### 환경 변수 주요 항목

| 변수 | 설명 |
|---|---|
| `LLM_PROVIDER` | `openai` / `anthropic` / `gemini` |
| `OPENAI_API_KEY` | API 키 (OpenAI 또는 호환 게이트웨이) |
| `GEMINI_API_KEY` | Google AI Studio 키 |
| `ANTHROPIC_API_KEY` | Anthropic API 키 |
| `LLM_MODEL_PRO` | 에이전트 진단·토론·심판용 모델 |
| `LLM_MODEL_OMNI` | 멀티모달(이미지·비디오 분석)용 모델 |
| `LLM_MODEL_FAST` | 빠른 작업(토론 라운드·댓글)용 모델 |
| `LLM_FREE_TIER_MODE` | `1` 설정 시 토론 라운드 생략 (무료 티어 대응) |
